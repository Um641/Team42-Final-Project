## 🏎️ 專案說明與系統架構
以下為工創期末循跡自走車原始碼。我們採用ESP32作為主控制器，並結合藍牙通訊模組，實現遠端起跑與監控功能。

### 1.硬體腳位配置 
* **動力系統 (L298N)：** * 左輪: `IN1=18`, `IN2=19`, `ENA=21`
  * 右輪: `IN3=23`, `IN4=17`, `ENB=16`
* **感測系統 (5路 TCRT5000)：** 左至右依序接於 `Pin 13, 33, 14, 27, 26`
* **安全與指示 (ASL & EBS)：** * RGB 指示燈: `R=5`, `G=4`, `B=22`
  * 緊急煞車按鈕 (EBS): `Pin 32` 

### 2.核心演算法與控制策略 (PD Control)
為了極大化賽道表現，我們的循跡邏輯採用了 **動態配速的 PD 控制器**，具備以下幾項亮點：
* **數位感測加權：** 將 5 顆感測器賦予權重 (+2, +1, 0, -1, -2)，並透過黑線感測數量取平均，精準計算出當前誤差值 (`current_error`)。
* **動態直線加速 / 彎道煞車：** 系統會隨時監控誤差值。當 `error <= 0.5` (車體居中) 時，基礎車速會不斷累加至極速 (`MAX_BASE_SPEED`)；一但偵測到彎道 (誤差變大)，系統會觸發 `DECEL_RATE` 瞬間降速，實現「入彎減速、出彎加速」的類賽車跑法。
* **極限過彎調校：** 加大轉向靈敏度 (`KP = 550`, `KD = 150`)，讓車體在銳角彎道時能以近乎原地旋轉的方式通過，有效克服 L298N 的靜摩擦力死區。
* **斷線記憶防呆 (Out of Control Protection)：** 若車體因車速過快完全脫離黑線，系統會記憶「最後一次的誤差方向 (`last_error`)」，並鎖定最大轉向極限強制找回黑線，若脫線超過 5 秒則自動觸發 EBS 保護機制。

### 3. 進階要求實作 (Advanced Requirements)
* **遠端藍牙通訊：** 使用 `BluetoothSerial`，不僅支援 APP 發送字元 'V' 觸發起跑，也能發送 'E' 執行遠端緊急煞車，並會回傳中文狀態訊息至手機端。
* **狀態燈號 (ASL) 確實連動：** 依據比賽規範，停車時亮綠燈 (SAFE)、藍牙待機時亮藍燈 (READY)、正式跑線時亮紅燈 (NAVIGATING)，全由狀態機嚴格管控。
